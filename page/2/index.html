Ionic框架结构

初始化Component的ViewController

  1. Ionic AppModule中，IonicModule.forRoot()方法传入的DeepLinkMetadata或是由@IonicPage在编译阶段解析得到的 IonicPageMetadata，将其统一转换成NavLink的对象的列表
  2. 由对应的名称在NavLink的对象列表中找到对应的NavLink
  3. 如果NavLink中存在link.component属性，那么由此属性进行解析，一般此属性为对应的component的构造方法，否则由link.loadChildren对应url动态加载（用到了SystemJs等相关的技术，然后使用Angular的技术）
  4. 通过页面所在模块中IonicPageModule.forChild()方法中参数拿到对应的component的构造方法
  5. 再将component传入ViewController中(注意此时component并未实例化，其实例化是在导航的页面中插入的时候，被Angular动态创建出来)
  6. 以ViewController为载体进行初始化。
  7. Angular动态创建组件
  8. 呈现到页面

Ionic中页面导航:

viewport是在网页中呈现组件结构的视窗

  1. NavController
    导航的控制器，导航功能的类的基类，实现了NavigationContainer的接口，在页面中都是以基类NavController的形式来使用导航类
  2. NavControllerBase
    继承于NavController，是所有真实的具有导航功能的类的基类
  3. Nav
    真实的导航类，维护了一个用来显示页面视图的viewport，以及具体的页面控制器的堆栈
  4. Tabs
    非真实导航类，因为其并不存在用来显示页面视图的viewport，也无具体的页面控制器的堆栈，只负责控制及管理其子级Tab的运作
  5. Tab
    真实的导航类，维护了一个用来显示页面视图的viewport，以及具体的页面控制器的堆栈，其运作受到Tabs的控制
  6. OverlayPortal
    覆盖层的入口，利用指令，指定某个html tag为页面视图的viewport

11. _viewAttachToDOM

正式执行动画
此处执行Ionic生命周期：ionViewWillEnter、ionViewWillLeave,ionViewWillUnload

GCD是在你程序中执行异步或者同步的任务的一种简单方式

  • 串行调度队列
    也被称为私有调度队列(private dispatch queue)，按照他们被加入队列的顺序，在同一时间只执行一个任务，当前被执行的任务在当前调度队列所管理的不同线程中同时执行，但是串行调度队列与串行调度队列直接是同时执行的
  • 并行调度队里
    也被称为全局调度队列（global dispatch queue）同时执行一个或者多任务，但是任务开始的顺序依旧是以其加入队列的顺序为准，当前被执行的任务在当前调度队列所管理的不同线程中同时执行，在某个时间点的任务的执行数目是一个取决于系统条件的变量

  • Main调度队列
    全局可得的执行在应用的主线程中的串行调度队列，此队列与程序的运行循环同步执行

  • 注意在block中使用大结构体或者其他的一些指针类型的变量的引用，可能会存在内存已经释放了，但是还在使用其的情况出现

  • 调度队列会拷贝加入其中的的block们，并且在执行结束时释放。也就是说你无需再胶乳到执行队列之前对其就行拷贝
  • 当你的block中需要创建一些Objective-C的对象时，最好是将题目放到@autorelease的块中让其管理其内存，尽管gcd有其自己的自动回收池，但是也不能保证其何时会不会泄露

objc/objc.h

objc/runtime.h

首先得知道，OC中一切皆是对象，类也是对象（或者可以称之为类对象），由一个类实例化来的对象叫“实例对象”，也就是说实例对象所属的类叫“类”，元类就是类对象所属的类，因此，实例对象是类的实例，类作为对象又是元类的实例。

那么既然一切皆是对象，那么元类也是对象，那元类对象的类又是什么呢？
元类对象的类叫根元类，也是NSObject的所属类（注意区分NSObject的实例对象的所属的类）而根元类的类即它本身

  • 实例对象：当我们在代码中new一个实例对象时，拷贝了实例所属的类的成员变量，但不拷贝类定义的方法。调用实例方法时，根据实例的isa指针去寻找方法对应的函数指针。

  • 类对象：是一个功能完整的对象。特殊之处在于它们是由程序员定义而在运行时由编译器创建的，它没有自己的实例变量（这里区别于类的成员变量，他们是属于实例对象的，而不是属于类对象的，类方法是属于类对象自己的），但类对象中存着成员变量与实例方法列表。

  • 元类对象：OC 的类方法是使用元类的根本原因，因为其中存储着对应的类对象调用的方法即类方法。其他时候都倾向于隐藏元类，因此真实世界没有人发送消息给元类对象。元类的定义和创建看起来都是编译器自动完成的，无需人为干涉。

        NSLog(@”cordova-plugin-wechat has been initialized. Wechat SDK Version: %@. APP_ID: %@.”, [WXApi getApiVersion], appId);
        }

        注：[self.commandDelegate settings]的值为platforms/ios/企业服务云/config.xml中的preference设置的对应的键值，这个文件也是真正的对于对应平台使用到的值
